## 🚀 Featured Projects

<details>
<summary><strong>⚡ DecentraVault — Decentralized Asset Management Protocol</strong></summary>

<br/>

A permissionless, non-custodial DeFi protocol enabling users to deploy multi-strategy yield vaults on Ethereum and L2 networks. Built with a security-first architecture and audited smart contracts using formal verification tooling.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | Solidity, Hardhat, Next.js 14, wagmi, ethers.js, Tailwind CSS, The Graph |
| **Scale** | 2,400+ wallet connections · $1.2M TVL (testnet simulation) · 15K+ transactions |
| **Performance** | Sub-200ms frontend loads · Gas-optimized contracts (avg 42K gas/tx) |
| **Security** | Reentrancy guards · Role-based access · Slippage protection · Multisig admin |
| **Impact** | End-to-end DeFi protocol covering vault strategy, UI, and subgraph indexing |
| **Repository** | [github.com/smriti-adhikari/decentravault](https://github.com/smriti-adhikari/decentravault) |

</div>

DecentraVault introduced a modular vault architecture where each yield strategy is an isolated, upgradeable contract. The protocol includes an on-chain governance module (Snapshot + Governor Bravo pattern), a referral system, and a custom subgraph for real-time analytics — delivering a production-grade dApp experience from contract layer to frontend.

<br/>

</details>

<details>
<summary><strong>🧠 NexusAI — Intelligent SaaS Platform with LLM Orchestration</strong></summary>

<br/>

A B2B SaaS platform enabling enterprises to deploy private LLM-powered workflows — document analysis, knowledge base Q&A, and automated report generation — over their own data without exposing it to external APIs.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | Next.js 14, FastAPI, LangChain, PostgreSQL, pgvector, Redis, Docker, AWS ECS |
| **Scale** | 3,200+ enterprise users · 500K+ documents indexed · 99.97% uptime |
| **Performance** | P95 query latency 340ms · RAG retrieval under 180ms · Auto-scaling ECS clusters |
| **Security** | JWT + RBAC auth · AES-256 data encryption · Private VPC deployment · SOC2-aligned |
| **Impact** | Reduced client document review time by 68% · $400K ARR within 10 months |
| **Repository** | [github.com/smriti-adhikari/nexusai](https://github.com/smriti-adhikari/nexusai) |

</div>

NexusAI features a multi-tenant architecture with isolated vector namespaces per organization using pgvector. The LangChain orchestration layer supports streaming responses, tool-calling agents, and custom embedding pipelines. The platform ships with a white-label dashboard, usage analytics, and a Slack integration for surfacing AI insights directly in team workflows.

<br/>

</details>

<details>
<summary><strong>🛒 QuantumCommerce — High-Performance E-Commerce Engine</strong></summary>

<br/>

A headless, composable e-commerce platform engineered for high-concurrency retail scenarios — flash sales, live commerce, and global storefronts — built on a microservices backbone with event-driven order orchestration.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | React, Node.js, Express, PostgreSQL, MongoDB, Redis, Kafka, Kubernetes, AWS |
| **Scale** | 85K+ concurrent users (peak load) · 1.4M+ orders processed · 12 microservices |
| **Performance** | P99 checkout latency 280ms · 99.995% availability · CDN cache hit rate 94% |
| **Security** | PCI-DSS compliant · Rate limiting · OWASP hardened · Stripe + Razorpay integration |
| **Impact** | Supported flash sales of 10K+ items sold in under 90 seconds without degradation |
| **Repository** | [github.com/smriti-adhikari/quantumcommerce](https://github.com/smriti-adhikari/quantumcommerce) |

</div>

QuantumCommerce separates the catalog, inventory, order, payment, and notification services into independently deployable units, each with its own data store following CQRS principles. Kafka powers the event bus for async order lifecycle management. The platform includes a real-time inventory reservation system that prevents overselling under extreme concurrent load, tested to 100K RPS via k6 load testing.

<br/>

</details>

<details>
<summary><strong>📊 PulseMetrics — Real-Time Engineering Observability Dashboard</strong></summary>

<br/>

An open-source, self-hosted observability platform for engineering teams — aggregating logs, traces, metrics, and deployment events into a unified, real-time dashboard with AI-powered anomaly detection and alerting.

<div align="center">

</div>

PulseMetrics is built around OpenTelemetry as the unified instrumentation standard, ingesting data from any language or framework via the OTLP protocol. The AI anomaly detection layer uses a lightweight time-series forecasting model trained per service to surface contextual alerts — reducing noise by 73% compared to static threshold alerts. It ships as a single `docker-compose up` for instant local deployment.
